# Architecture Overview

## Product Split

### Flutter App

- Frontend: Flutter dashboard for live demo use on mobile, tablet, or desktop.
- Backend: lightweight Node mock service in `apps/flutter_app/backend/`.
- AI agent: prompt and fallback playbook in `apps/flutter_app/agent/`.

### Web App

- Frontend: Next.js App Router UI in `apps/web_app/src/components/`.
- Backend: API routes in `apps/web_app/src/app/api/`.
- AI agent: orchestration logic in `apps/web_app/src/lib/agent/`.

## Core Domains

- `ProspectSignal`: the salon ranking unit.
- `OutreachDraft`: the personalized B2B outreach unit.
- `ContentAsset`: the AI-search and content production unit.
- `ReviewItem`: the human-in-the-loop approval unit.
- `AiErrorState`: the visible fallback and escalation unit.

## Operating Model

1. Ingest luxury salon signals.
2. Score for fit and retail upside.
3. Draft guarded outreach.
4. Generate brand-safe content.
5. Route uncertain outputs to human review.
6. Show AI incidents with fallback guidance.

## Why The Yellow Dot Exists

The yellow dot is the developer marker for AI-controlled surfaces. If the dot is visible, the current card, screen, or workflow is either:

- generated by AI
- powered by mocked AI for demo purposes
- in fallback mode
- pending human review
